# Excel Analyst Bot

Telegram-бот для анализа Excel и CSV-файлов. Он умеет строить автоматические отчеты, рассчитывать бизнес-KPI, собирать пользовательские графики, искать аномалии, делать прогнозы и хранить историю анализов.

## Возможности

- загрузка `.xlsx`, `.xls` и `.csv` файлов;
- автоматический EDA;
- определение типов и бизнес-колонок;
- бизнес-анализ продаж, маркетплейсов, склада и клиентской базы;
- сравнение двух отчетов;
- поиск аномалий;
- прогнозирование продаж;
- UMAP-preview сразу после загрузки файла;
- отдельный UMAP-анализ;
- PCA / UMAP / t-SNE;
- кластеризация и автообъяснение сегментов;
- конструктор графиков;
- PDF-отчеты;
- история таблиц, отчетов и графиков через SQLite;
- Streamlit / Plotly dashboard;
- команда `/history`.

## Главное меню

После `/start` бот показывает 4 кнопки:

- `Мои таблицы`;
- `Мои отчеты`;
- `Мои графики`;
- `На главную`.

Кнопка загрузки файла находится в стартовом сообщении.

## Сценарий работы

1. Отправьте `/start`.
2. Нажмите кнопку загрузки файла.
3. Отправьте Excel или CSV документом.
4. Бот покажет UMAP-preview и меню действий по этому файлу.
5. Выберите нужный анализ, график или отчет.
6. После результата можно продолжить работать с тем же датасетом без повторной загрузки.

## Конструктор графиков

Конструктор позволяет вручную собрать визуализацию по выбранным колонкам.

Доступны:

- линейный график;
- столбчатая диаграмма;
- горизонтальная bar-диаграмма;
- гистограмма;
- круговая диаграмма;
- scatter plot;
- boxplot;
- heatmap корреляций;
- area chart;
- мультиграфик;
- комбинированный график.

Формат результата:

- `PNG`;
- `PDF`.

После построения можно:

- построить еще один график по тому же файлу;
- вернуться к действиям по файлу;
- загрузить новый файл;
- собрать несколько графиков в один PDF.

## ML-анализ структуры данных

После загрузки файла доступны:

- `PCA / UMAP / t-SNE`;
- `Кластеризация`;
- `PowerPoint-отчет`.

Кратко:

- PCA показывает общую линейную структуру;
- UMAP помогает увидеть локальные группы;
- t-SNE полезен для локальных соседств;
- KMeans ищет компактные сегменты;
- DBSCAN ищет плотные группы и шум;
- кластеры нужно интерпретировать осторожно.

## История

Бот сохраняет:

- загруженные таблицы;
- PDF-отчеты;
- графики из конструктора;
- UMAP и ML-результаты.

Историю можно посмотреть через кнопки меню или команду `/history`.

## Запуск локально

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
$env:BOT_TOKEN="ваш_токен_бота"
python bot.py
```

## Dashboard

Отдельное приложение Streamlit:

```powershell
streamlit run dashboard/app.py
```

## Переменные окружения

- `BOT_TOKEN` - токен Telegram-бота;
- `MAX_FILE_SIZE_MB` - максимальный размер загружаемого файла;
- `TEMP_DIR` - директория для временных файлов;
- `IMAGE_DIR` - директория для изображений;
- `OUTPUT_DIR` - директория для артефактов анализа.

## Примечания

- временные файлы лежат в `temp/` и не коммитятся;
- результаты и кэш лежат в `outputs/` и тоже не коммитятся;
- PDF сейчас является основным форматом для отчетов и пользовательских результатов;
- интерактивные карты и внутренние представления используются только там, где это нужно для логики проекта.
